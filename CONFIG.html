Production Configuration Guide

🚀 Deployment Instructions

1. GitHub Pages Setup

Repository Settings → Pages
- Source: Deploy from branch
- Branch: main (or master)
- Folder: / (root) - GitHub Pages will serve Kidney-Classifier/index.html

Alternative: Use /docs folder
- Move Kidney-Classifier contents to docs/ folder
- Set folder to /docs in Pages settings

2. GitHub Actions Workflow

Already configured in .github/workflows/pages.yml

The workflow automatically:

  • Listens for pushes to main/master
  • Uploads Kidney-Classifier/ as artifact
  • Deploys to GitHub Pages
  • Provides public URL

3. Environment Configuration

File: Kidney-Classifier/index.html
Line: ~2134

Update API endpoint:
const HF_API = 'YOUR_API_ENDPOINT_HERE';

Current endpoint (example):
https://thiyagu2004-kidney-classifier.hf.space/predict

4. Build & Deployment

Step 1: Make changes locally
$ git add .
$ git commit -m "description of changes"

Step 2: Push to GitHub
$ git push origin main

Step 3: Monitor deployment
- Go to GitHub Actions tab
- Watch pages.yml workflow
- Wait for ✅ completion

Step 4: Access your site
https://YOUR_USERNAME.github.io/kidney-classifier-frontend/Kidney-Classifier/

🚨 Troubleshooting

Pages not showing?

1. Check Settings → Pages
2. Verify branch and folder
3. Check GitHub Actions for errors
4. Clear browser cache
5. Wait 1-2 minutes for DNS propagation

Images not loading?

1. Verify file paths (use relative paths)
2. Check file names (case-sensitive)
3. Ensure images are in same folder
4. Check browser console for errors

API errors?

1. Verify API endpoint URL
2. Check backend is running
3. Verify CORS headers
4. Check browser console for details
5. Test API with curl/Postman
